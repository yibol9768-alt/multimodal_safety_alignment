% EMNLP/ARR mandatory limitations section. Bulleted for clarity.

\begin{itemize}
    \item \textbf{Scheme-specific applicability.} \method{}/\method{}+ targets VLAs that encode discrete actions by overwriting an LLM sub-vocabulary. It is not directly applicable to continuous-action VLAs (e.g., \textsc{OpenVLA-OFT}, \textsc{$\pi_0$}). We verify this by including \textsc{OpenVLA-OFT} as a scheme-specific negative control (Sec.~\ref{sec:analysis:oft}).
    \item \textbf{Shared pretraining assumption.} Cross-model transplant relies on the target VLA's backbone and the external aligned LLM sharing pretraining initialization. We do not evaluate transplant across different pretraining families (e.g., Llama-based VLA target with Qwen-aligned source); whether this works is left to future work.
    \item \textbf{First action token under base \method{}.} The base \method{} variant (action-only target) does not fire during the prefill step that generates $a_1$, because at that point no action token ids are present in the input. \method{}+ addresses this via dual-phase injection (Sec.~\ref{sec:method:dualphase}), but the text-side $\alpha_{\text{text}}=0.3$ injection is weaker than the action-side injection, so $a_1$ receives less steering pressure than $a_2\ldots a_7$ in the current implementation.
    \item \textbf{Simulated evaluation only.} Our evaluation is in simulation (\textsc{LIBERO}, \textsc{AI2-THOR}). Real-robot execution introduces distribution shift we do not address. A defense that triggers ``zero-motion'' in simulation may still be actuating unsafe behavior during the time it takes the real controller to receive a stop signal.
    \item \textbf{Attack coverage.} We evaluate instruction-level and patch-based attacks; we do not test novel physical adversarial perturbations, material-level camouflage, or multi-step adversarial plans. The adaptive attack in Sec.~\ref{sec:exp:adaptive} is an upper bound on white-box gradient-based attacks under $L_\infty$ constraints only.
    \item \textbf{Zero-motion proxy for refusal.} Our HRR-strict metric uses a zero-motion action window as a stand-in for ``the robot stopped''. Some refusals may take the form of alternative safe actions (e.g., placing the knife down) which our metric does not credit. HRR-partial partially alleviates this; a full rollout-level evaluation with human judges is deferred to camera-ready.
    \item \textbf{Content-adaptive $\alpha$ introduces a small auxiliary model.} The $g_\theta$ head (roughly $500\mathrm{K}$ parameters) is trained on labeled harmful/benign pairs, which means the adaptive variant of \method{}+ is not strictly training-free; only the rank-1 and rank-$k$ variants are. We disclose training cost and data for $g_\theta$ in Appendix.
    \item \textbf{Safety--usefulness frontier.} \method{}+ reduces harmful action rate at some cost to benign task success. We report BSR and HRR separately (rather than only BRR) to make this trade-off visible.
\end{itemize}
